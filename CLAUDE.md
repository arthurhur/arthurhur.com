# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A single-page personal portfolio site for Arthur Hur, served as static files from GitHub Pages. There is no build step, package manager, framework, test suite, or linter — just `index.html`, one stylesheet, one script, fonts, and images.

## Develop & deploy

- Preview by opening `index.html` directly — all asset paths are relative (`css/…`, `img/…`, `../fonts/…`), so CSS, fonts, and the per-row background stills all resolve under `file://`. For production-like behaviour (and so the YouTube/Vimeo player APIs and the crossorigin font preload act as they do live), serve the folder instead: `python3 -m http.server`, then visit `http://localhost:8000`.
- Deployment is automatic: the live site at **arthurhur.com** (set by `CNAME`) is published by GitHub Pages from the `main` branch. Pushing to `main` ships to production — there is no staging.

## Architecture

The one non-obvious mechanism is the **hover-to-swap background**. Two stacked `<div class="bg-layer">` elements crossfade: `setupBackground` in `js/script.js` paints the incoming still on the hidden layer, fades it in, then flips which layer is "front" (double-buffering avoids a flash). The resting background is the `--bg-default` custom property — currently `none`, so the warm `--bg` colour shows when nothing is open.

Each portfolio row is a `<button class="row" data-bgimg="img/bg-NAME.webp">` whose still becomes the backdrop. On pointer devices, hovering/focusing a row previews its still and leaving settles back ~200ms later to whatever is *pinned*; the stills are warmed off the critical path after first paint (`preloadWhenIdle`, batched via `requestIdleCallback`) so the first hover is instant without delaying load. On touch (no hover), `setupScrollReveal` fades in the still for whichever row is centred in the viewport, preloading each via `IntersectionObserver` so a phone isn't fetching every still up front. Opening a project **pins** its still as the resting background (so it stays while you read the panel); closing unpins back to the default.

Because of this, **adding or changing a portfolio entry is a two-part change** that must stay in sync:
1. The `<button class="row" … data-bgimg="img/bg-NAME.webp">` (plus its sibling `<div class="panel">…</div>`) in `index.html`
2. The corresponding `img/bg-NAME.webp` file

`setupBackground` selects `main [data-bgimg]` generically, so any row with a valid `data-bgimg` works automatically — no per-row wiring. There is no longer a default background *image* to keep in sync: `--bg-default` is `none`.

A related mechanism is the **film player**. Every YouTube film plays through YouTube's IFrame Player API (loaded lazily) with `controls=0` via `mountYouTube` in `js/script.js`, so it stays clean — `controls=0` is the one way to suppress the control bar, since any seek/replay re-arms it. An autoplaying Vimeo lead is wrapped by Vimeo's Player API via `mountVimeo` with a one-time tap-to-unmute overlay that then hands off to Vimeo's native controls for pause/scrub/fullscreen (background mode hides chrome but is non-interactive, and `controls=0` isn't reliably honored); a *clicked* Vimeo stays a plain native-controls iframe and plays once. At most one film per panel autoplays (muted): the first that *wants* to. A film's `data-autoplay="on"`/`"off"` attribute is an explicit toggle that overrides the positional default (a film that *leads* its panel — nothing rendered above it — autoplays). Every other film waits for a click and then plays with sound. An autoplaying film loops as ambient background (YouTube via an `ENDED → seek 0 + play` handler we own; Vimeo via `loop=1`); a clicked film plays once. Because `controls=0` removes the native volume button, a custom sound control (`buildSoundControls`) overlays each YouTube player: a transparent layer where a click anywhere unmutes (no-op once unmuted), plus a speaker toggle (the only way to mute back, while playing). It takes a small provider-agnostic `{ mute, unmute, isPlaying }` adapter. A leading YouTube film starts muted, a clicked one starts unmuted; Vimeo instead uses the one-time gate above and then its own controls. A session-wide `soundOn` preference carries the choice across films: once any film is unmuted (the overlay/speaker or the Vimeo gate), later leads open unmuted too — they still autoplay muted to satisfy autoplay policy, then unmute on ready (YouTube `unMute()`, Vimeo `setMuted(false)` with no gate); muting via the YouTube speaker carries the silence forward. Only one film plays at a time — when any API player starts, an `activePlayers` registry pauses the others (a clicked Vimeo, being a plain iframe, isn't tracked). Both APIs fall back to a plain iframe if they can't load. Film slots default to a 16:9 frame; a non-16:9 film (e.g. old SD music videos) would otherwise pillarbox. An autoplaying Vimeo lead auto-fits — `mountVimeo` reads the true dimensions via `getVideoWidth`/`getVideoHeight` and sets the slot's `aspect-ratio`. For anything that can't self-report (a clicked Vimeo, a non-16:9 YouTube), a `data-ratio="4/3"` (or `"4:3"`) attribute on the slot pins the frame and overrides the auto-fit. YouTube Shorts are a built-in special case: `videoEmbed` accepts `youtube.com/shorts/ID` (also `/embed/ID`, `/live/ID`, where the id lives in the path rather than a `?v=` param) and flags it `short`, so the slot auto-frames 9:16 — still overridable by `data-ratio`.

A third mechanism is the **accordion panels**. Each `.row` button is immediately followed by its own `<div class="panel">…</div>`; `setupAccordion` wires that sibling, and clicking a row toggles it open while collapsing any other open row (single-open) and stopping that row's films (`unloadEmbeds`). Opening pins the row's `data-bgimg` still as the resting background and lazy-mounts the panel's films (`loadPanelEmbeds`); closing unpins and unloads. When a row opens while another is open *above* it, `holdRowAtTop` clamps the clicked row at the top of the viewport so it doesn't sail off-screen as the page above it collapses. A panel body is an ordered flow of blocks, authored in any order — prose `<p>`, a film `<div class="media media--video" data-href="…">`, an image `<figure class="figure"><img src="img/NAME.webp" loading="lazy" decoding="async" width height>…</figure>`, a **local looping clip** (former GIFs, now MP4) `<figure class="figure"><video class="gifv" width height muted loop playsinline preload="none" poster="img/NAME-poster.webp"><source data-src="img/NAME.mp4" type="video/mp4"></video>…</figure>`, or an external `<a class="watch">` link (the HTML comment above the first `<ul class="works">` is the legend). Clips are lazy by design — the `<source>` carries only `data-src`, so nothing loads until `loadPanelEmbeds` wires the real `src` and plays it (muted, looping) on open, and `unloadEmbeds` pauses it on close; `prefers-reduced-motion` leaves just the poster. Media can sit in a `.media-grid--justified` row: `setupJustifiedMedia` reads each item's intrinsic `width`/`height` (img or video) into an `--ar` custom property and the row uses `flex-grow: var(--ar)`, so everything resolves to the same height (bottoms and captions aligned) while the row still fills its width.

## Conventions

- Theming is driven by CSS custom properties in `:root` (`css/styles.css`): colours `--bg`, `--fg`, the `--muted*` greys, `--accent`, `--rule`, `--hover-tint`; the resting backdrop `--bg-default` (currently `none`); and the type stacks `--serif`/`--sans`/`--mono`. Change these rather than hardcoding.
- Fonts are **self-hosted, latin-subset `woff2`** in `fonts/`, declared via `@font-face` in `css/styles.css` (no CDN): **Newsreader** (serif) as one variable file preserving its `opsz`/`wght` axes, **Schibsted Grotesk** (sans) and **IBM Plex Mono** (mono) as static 400 + 500. Only what the CSS uses is shipped — weights 400/500, roman only, no italics. Each family also has a **metric-matched fallback** face (`… Fallback`, a local system font re-shaped with `size-adjust`/`ascent-override`/etc., first in each stack after the real font) so the font swap on load causes no layout shift; the override numbers are computed from font metrics (capsize method) — recompute, don't hand-tune, if you swap a font. To add a weight or glyph, re-subset from a full source with `pyftsubset … --unicodes=<latin + extra codepoints> --flavor=woff2`. Note the standard latin subset omits `→` (U+2192), which is added explicitly; Newsreader has no `→` glyph at all, so that one character falls back to a system serif.
- Media is optimised for fast loading: background stills and panel images are **WebP** (stills capped at 2560px), animations are **MP4 `<video class="gifv">`** rather than GIFs (~10× smaller; encode with ffmpeg, poster from a first-frame WebP), and every panel `<img>`/`<video>` carries intrinsic `width`/`height` so there's no layout shift. The social `og:image`/`twitter:image` stays a JPEG (`img/og-cover.jpg`) since some scrapers don't read WebP. Don't commit raw GIFs or multi-MB 4K stills — re-encode first (originals for this pass were archived outside the repo).
- Google Analytics (gtag, ID `G-01V10S2M76`) loads inline in `<head>` — keep that block intact when editing the head.
